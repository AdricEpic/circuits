  5  """Event framework with a Component architecture 
  6   
  7  circuits is an event-driven framework with a focus on Component Software 
  8  Architectures where System Functionality is defined in Components. 
  9  Components communicate with one another by propagating events throughout 
 10  the system. Each Component can react to events and expose events to other 
 11  parts of the system Components are able to manage their own events and 
 12  can also be linked to other Components. 
 13   
 14  circuits has a clean architecture and has no external dependencies on any 
 15  other library. It's simplistic design is unmatchable but yet delivers a 
 16  powerful framework for building large, scalable, maintainable applications 
 17  and systems. circuits was a core integral part of the 
 18  [http://trac.softcircuit.com.au/pymills pymills] library developed in 2006 
 19  and was partly inspired by the [http://trac.edgewall.org Trac] architecture. 
 20   
 21  Simple Example: 
 22   
 23  >>> from circuits.core import Component, Event, Manager 
 24  >>> 
 25  >>> class Hello(Component): 
 26  ...   def hello(self): 
 27  ...      print "Hello World!" 
 28  >>> manager = Manager() 
 29  >>> manager += Hello() 
 30  >>> manager.push(Event(), "hello") 
 31  >>> manager.flush() 
 32  Hello World! 
 33  """
